## 📊 架构图预览

系统包含以下7个核心层:

```
┌─────────────────────────────────────────────────────┐
│  第1层: 硬件输入层 (Nintendo Joy-Con)                │
│  🔵 蓝色 - 陀螺仪+加速度计+按键 @ 100Hz              │
└──────────────────┬──────────────────────────────────┘
                   │ 蓝牙数据流
                   ↓
┌─────────────────────────────────────────────────────┐
│  第2层: 姿态解算层 (joycon-robotics)                 │
│  🟠 橙色 - 互补滤波+低通滤波 → [x,y,z,r,p,y]        │
└──────────────────┬──────────────────────────────────┘
                   │ 6自由度位姿
                   ↓
┌─────────────────────────────────────────────────────┐
│  第3层: ROS2 适配层 (Joycon 遥操作节点)              │
│  🟢 绿色 - 坐标转换+限制检查 → PoseStamped          │
└──────────────────┬──────────────────────────────────┘
                   │ /target_pose
                   ↓
┌─────────────────────────────────────────────────────┐
│  第4层: 运动规划层 (MoveIt2)                         │
│  🟣 紫色 - 逆运动学+路径规划+碰撞检测               │
└──────────────────┬──────────────────────────────────┘
                   │ 关节轨迹
                   ↓
┌─────────────────────────────────────────────────────┐
│  第5层: 机器人控制层 (Parol6 驱动)                   │
│  🩷 粉红 - 轨迹跟踪+电机控制 → USB 串口             │
└──────────────────┬──────────────────────────────────┘
                   │ 电机指令
                   ↓
┌─────────────────────────────────────────────────────┐
│  第6层: 硬件执行层 (Parol6 机械臂)                   │
│  🔴 红色 - 6轴机械臂 (400mm/1kg/0.08mm)             │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  第7层: 数据采集层 (轨迹记录器 + 相机)               │
│  🟡 黄色 - HDF5数据集+图像流 @ 100Hz                │
└─────────────────────────────────────────────────────┘
```

**总延迟**: <50毫秒（端到端）
**控制频率**: 100Hz
**数据采集**: 100Hz（支持 ACT/扩散策略）

## 🎯 核心功能模块

### 输入处理
| 模块 | 文件 | 功能 |
|------|------|------|
| Joy-Con 连接 | joycon.py | 蓝牙设备管理 |
| 姿态估计 | gyro.py | 互补滤波+漂移补偿 |
| 按键事件 | event.py | 按键映射和处理 |

### ROS2 接口
| 模块 | 文件 | 功能 |
|------|------|------|
| 遥操作节点 | joycon_teleop_node.py | 主控制逻辑 |
| 配置管理 | parol6_config.yaml | 参数配置 |

### 运动控制
| 模块 | 包名 | 功能 |
|------|------|------|
| MoveIt2 规划 | moveit_ros_move_group | 逆运动学+路径规划 |
| 机器人驱动 | parol6_driver | 硬件接口 |

### 数据采集
| 模块 | 文件 | 功能 |
|------|------|------|
| 轨迹记录 | trajectory_recorder.py | HDF5 数据集生成 |
| 相机采集 | camera_node.py | RGB/深度图像 |
| 数据管理 | dataset_manager.py | 数据集组织 |

## ⚙️ 关键配置参数

### Parol6 工作空间
```yaml
工作空间:
  x: [0.05, 0.40]  # 米
  y: [-0.35, 0.35]
  z: [0.05, 0.50]

初始位姿: [0.25, 0.0, 0.2, 0.0, 0.0, 0.0]
```

### Joycon 映射参数
```yaml
欧拉角反转: [1, -1, 1]
方向反转: [-1, 1, 1]
偏移欧拉角: [0, -3.14159, 0]
俯仰加倍: false
纯XZ模式: false
```

### 控制参数
```yaml
最大速度缩放: 0.3
最大加速度缩放: 0.3
自由度速度: [1.0, 1.0, 1.0, 1.2, 1.2, 1.5]
```

### 数据采集
```yaml
录制频率: 100  # Hz
图像分辨率: [640, 480]
保存格式: "hdf5"
```

## 🔄 数据流说明

### 实时控制流（主线）
```
Joy-Con → joycon-robotics → ROS2节点 → MoveIt2 → Parol6驱动 → 机械臂
(100Hz)     (姿态解算)      (坐标转换)   (路径规划)   (电机控制)    (执行)
```

### 反馈流（虚线）
```
机械臂 → Parol6驱动 → ROS2节点（工作空间检查）
                   → MoveIt2（碰撞检测）
```

### 数据采集流（虚线）
```
ROS2节点 ──┐
Parol6驱动 ─┼→ 轨迹记录器 → 数据集管理器 → HDF5
相机 ────┘
```

## 🐛 故障排除

### 常见问题

#### 问题1：Joy-Con 无法连接？
**解决方案**:
1. 重新运行 `make install`
2. 删除蓝牙配对信息后重新配对
3. 检查 Ubuntu 蓝牙服务状态
4. 参考 `architecture_guide.md` 的连接章节

#### 问题2：姿态漂移严重？
**解决方案**:
1. 长按 Home 键重新校准
2. 检查 `lowpassfilter_alpha_rate` 参数
3. 确保 Joy-Con 电量充足
4. 避免磁场干扰

#### 问题3：MoveIt2 规划失败？
**解决方案**:
1. 检查目标位姿是否在工作空间内
2. 增加 `planning_time` 参数
3. 调整 `max_velocity_scaling`
4. 检查碰撞场景配置

#### 问题4：数据采集卡顿？
**解决方案**:
1. 降低图像分辨率
2. 使用 SSD 存储
3. 检查 CPU/内存占用
4. 考虑使用 ROS2 bag 录制
